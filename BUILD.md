# Build Notes

## Standard build and test flow

Use the unified harness to run the standard validation pipeline:

```bash
python scripts/run_tests.py
```

The harness executes the following discrete steps in order:

1. RPC binding generation via `python scripts/generate_rpc_bindings.py`.
2. DB namespace generation via `python scripts/generate_db_namespace.py`.
3. Route registry generation via `python scripts/generate_nav_pages.py`.
4. Frontend linting via `npm run lint`.
5. Frontend type checking via `npm run type-check`.
6. Frontend tests via `npx vitest run` (attempts coverage first, then retries without coverage if needed).
7. Backend tests via `pytest` (uses `pytest-cov` coverage reporting when available, otherwise runs without coverage).

When you need a frontend production build outside the harness, run the generator scripts before running the frontend build.

## Docker build intent

The Dockerfile is a multi-stage build:

- The **builder** stage installs Node + Python, installs Python requirements, generates RPC bindings, DB namespaces, and the route registry, installs frontend dependencies, and produces the frontend build artifacts.
- The **runtime** stage installs Python + OS dependencies, copies the `static/` output from the builder stage, installs Python requirements, and then copies the repo for runtime execution.

Generator scripts (`generate_rpc_bindings.py`, `generate_db_namespace.py`, and `generate_nav_pages.py`) run in the builder stage before `npm run build`, ensuring the frontend build has the generated inputs it requires. The frontend build expects the generated artifacts plus the `frontend/` sources and `package.json`/lock data to be present in the build context.

## Generated DB namespace

`frontend/src/db/namespace.ts` is generated by `python scripts/generate_db_namespace.py`. Do not edit or commit this file; regenerate it as part of build or dev steps.

## Generated RPC namespace

Do not commit the output of `python scripts/generate_rpc_namespace.py`. Regenerate it as needed instead of checking it into source control.

## Generated Route Registry

`frontend/src/routes/registry.ts` is generated by `python scripts/generate_nav_pages.py`
from the `frontend_pages` database table. Do not edit the generated file; update the
database table and regenerate. In environments without database access (e.g., Codex CI),
generation is skipped gracefully.
