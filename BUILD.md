# Build Notes

## Standard build and test flow

Use the unified harness to run the standard validation pipeline:

```bash
python scripts/run_tests.py
```

The harness executes the following discrete steps in order:

1. RPC binding generation via `python scripts/generate_rpc_bindings.py`.
2. DB namespace generation via `python scripts/generate_db_namespace.py`.
3. Frontend linting via `npm run lint`.
4. Frontend type checking via `npm run type-check`.
5. Frontend tests via `npx vitest run` (attempts coverage first, then retries without coverage if needed).
6. Backend tests via `pytest` (uses `pytest-cov` coverage reporting when available, otherwise runs without coverage).

When you need a frontend production build outside the harness, use the `dev.cmd` flow so that RPC bindings and DB namespaces are generated before running the frontend build.

## Docker build intent

The Dockerfile is a multi-stage build:

- The **builder** stage installs Node + Python, installs Python requirements, generates RPC bindings and DB namespaces, installs frontend dependencies, and produces the frontend build artifacts.
- The **runtime** stage installs Python + OS dependencies, copies the `static/` output from the builder stage, installs Python requirements, and then copies the repo for runtime execution.

Generator scripts (`generate_rpc_bindings.py` and `generate_db_namespace.py`) run in the builder stage before `npm run build`, ensuring the frontend build has the generated inputs it requires. The frontend build expects the generated artifacts plus the `frontend/` sources and `package.json`/lock data to be present in the build context.

## Windows dev workflow (`dev.cmd`)

Use `dev.cmd` for Windows development workflows.

Supported commands:

- `dev.cmd generate` → Pull latest from GitHub, then run RPC binding generation and DB namespace generation.
- `dev.cmd start` → Install Python deps, run generators, run frontend lint/type-check/test/build, run backend tests, then start Uvicorn with reload.
- `dev.cmd fast` → Install Python deps, run generators, then start Uvicorn with reload (skips frontend tasks and tests).
- `dev.cmd test` → Pull latest, install Python deps, run generators, run frontend lint/type-check/test/build, then run backend tests.
- `dev.cmd restart` → Pull latest, install Python deps, run generators, run frontend lint/type-check/test/build, run backend tests, then start Uvicorn with reload.
- `dev.cmd refast` → Pull latest, install Python deps, run generators, run frontend lint/type-check/test/build, run backend tests, then start Uvicorn with reload (skips `npm ci`).

## Generated DB namespace

`frontend/src/db/namespace.ts` is generated by `python scripts/generate_db_namespace.py`. Do not edit or commit this file; regenerate it as part of build or dev steps.

## Generated RPC namespace

Do not commit the output of `python scripts/generate_rpc_namespace.py`. Regenerate it as needed instead of checking it into source control.
